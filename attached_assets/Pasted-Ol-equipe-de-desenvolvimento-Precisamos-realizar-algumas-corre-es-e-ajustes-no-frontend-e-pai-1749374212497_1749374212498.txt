Olá, equipe de desenvolvimento,

Precisamos realizar algumas correções e ajustes no frontend e painel administrativo do nosso sistema. Abaixo estão os pontos que devem ser tratados com detalhes, incluindo localização exata dos elementos no código e exemplos de trechos HTML/JSX para ajudar na implementação:

1. Remover a aba "Configurações" do painel administrativo

   * Local: componente <code>client/src/components/header.tsx</code>
   * Detalhes: no menu de navegação do admin, remova o <li> contendo <code>"Configurações"</code>.
   * Exemplo:

     ```jsx
     {/* Antes */}
     <ul>
       <li><Link to=\"/admin/settings\">Configurações</Link></li>
       <li><Link to=\"/admin/submissions\">Submissões</Link></li>
     </ul>
     {/* Depois */}
     <ul>
       <li><Link to=\"/admin/submissions\">Submissões</Link></li>
     </ul>
     ```

2. Inserir botão de confirmação na página /admin em "Painel Administrativo > Config. Site"

   * Local: arquivo <code>client/src/pages/admin/settings.tsx</code>
   * Problema: não existe botão para confirmar atualização dos dados; atualmente as alterações são enviadas automaticamente ao digitar.
   * Solução: adicionar um <button id="confirm-update-btn">Confirmar</button> logo após os campos do formulário.
   * Exemplo:

     ```jsx
     {/* Dentro do form */}
     <Input id="site-title" value={siteTitle} onChange={handleTitleChange} />
     <button id="confirm-update-btn" onClick={handleConfirmUpdate}>Confirmar</button>
     ```

3. Remover redirecionamento da imagem do rodapé para a página admin

   * Local: componente <code>client/src/components/footer.tsx</code>
   * Problema: a imagem de logo no rodapé está dentro de um <Link to="/admin">, causando redirecionamento.
   * Solução: remover o <Link> e manter apenas a <img> com o <alt> apropriado.
   * Exemplo:

     ```jsx
     {/* Antes */}
     <Link to="/admin">
       <img id="footer-logo" src={logoSrc} alt="Logo Admin" />
     </Link>
     {/* Depois */}
     <img id="footer-logo" src={logoSrc} alt="Logo Admin" />
     ```

4. Desabilitar atualização automatica na aba "Config. Site"

   * Local: <code>client/src/pages/admin/settings.tsx</code>
   * Problema: o state do formulário chama a função de update na API em cada onChange.
   * Solução: alterar o form para que as chamadas
     de API ocorram apenas dentro da função handleConfirmUpdate, removendo as triggers de onChange.
   * Exemplo:

     ```jsx
     // Antes
     <Input onChange={(e) => updateSite({ title: e.target.value })} />

     // Depois
     <Input onChange={handleTitleChange} />
     // e dentro de handleConfirmUpdate:
     const handleConfirmUpdate = () => {
       updateSite({ title: siteTitle, ... });
     };
     ```

Por favor, sigam essas instruções detalhadas e verifiquem se todas as referências de IDs correspondem corretamente aos elementos no JSX/HTML. Caso haja outras dependências ou testes afetados, atualizem também os arquivos de rota e possíveis mocks de testes unitários.
